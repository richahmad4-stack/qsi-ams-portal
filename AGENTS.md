# AMS Agent Instructions

At the end of every working session, commit all changes with a message describing what was built or changed this session, and push to origin main. Also update PROGRESS.md before committing.

Before staging or committing, keep `.env`, `vendor/`, runtime writable data, local uploads, logs, sessions, caches, and local database files out of Git. Never commit live passwords, API keys, SMTP secrets, client data, personal data, or real audit records.
